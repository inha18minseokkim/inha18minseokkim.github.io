### 1. 내 보유자산 연동 (마이데이터)

![이미지](/assets/images/Pasted%20image%2020260226143818.png)

**Q. 내 보유자산을 어떻게 수신하나? 케이뱅크는 아직 마이데이터 사업자가 없는데?**

- → 카카오페이 제휴형 마이데이터로 수신 예정 (계정계 적재 없음). 가상자산은 업비트 API 연동 방식으로 경영테크팀 담당 예정.
- 제휴형 마이데이터 방식이므로 채널에서 웹뷰 형태로 표시하는 것이며, 데이터가 케이뱅크 앱 내부로 직접 유입되지는 않을 것으로 판단.
- ⚠️ 컴플라이언스 이슈: 내 자산 현황 화면에서 증권과 가상자산을 함께 노출할 수 없다는 규정 확인 필요.

### 2. 공모주 API 정합성 (160 연동)

![이미지](/assets/images/Pasted%20image%2020260226143824.png)

**Q. 해당 API를 160 시스템을 통해 수신하겠다고 하는데, 공모주 메이트 서비스와 데이터 정합성 이슈가 발생하지 않을지? 아니면 160에서 화면 뷰를 직접 제공하고 채널에서 호출만 하는 방식인지?**

- 연합인포맥스와 160에서 수신하는 데이터가 일치해야 함.
- 160 API를 통해 직접 수신하는 경우, 연합인포맥스 데이터와 정합성 검증 예정.
  - 참고: 최근 연합인포맥스에서 미래에셋스팩 4호 청약 직전까지 데이터가 누락된 사례가 있었음 (연합인포맥스 DB에도 없어서 자체 제보 후 인지됨).

### 4. GPT 퀴즈 기능

![이미지](/assets/images/Pasted%20image%2020260226143848.png)

GPT를 활용한 퀴즈 생성 방향은 공감하나, 구현 방식에 대한 논의 필요.

1. **파인튜닝 여부**: 예제 데이터셋 기반의 파인튜닝이 필요할 수 있음.
2. **Hallucination 대응 방안**:
   - 오답을 허용(fault-tolerant)하는 방식으로 운영할 것인지.
   - 퀴즈 기준 정보를 직접 관리할 것인지.
     - 직접 관리 시 매일/매주 담당자의 수동 관리 필요 → BM 측 운영 준비 여부 확인 필요.
     - 관리자 화면 개발도 함께 검토 필요.
3. **저작권 문제**:
   - 파인튜닝 시 예제 데이터셋 필요 → 데이터셋 출처 및 저작권 확인 필요.
   - 자체 제작 시 리소스 과다 발생 가능성.

2. 코인 데이터 처리 방안:
   - 코인 시가총액 지수 포함 여부 결정 필요.
   - 경영테크팀에서 업비트 API 연동 시, 지수 관련 데이터는 별도 제공 예정.
   - 데이터 출처가 다른 경우 매시업(mashup) 필요 여부 검토.
- → 나스닥 데이터는 딥서치에서 제공 가능. 코인 데이터는 경영테크팀에서 수신 후, 채널에서 두 출처를 각각 호출하는 방식으로 처리 예정.
