## 📌 이슈 요약

- `include_trading_halted = true` → 404 Symbol Not Found 발생
- 딥서치 측 초기 안내: 거래 불가 종목만 유효한 옵션
- 최종 확인 결과: **모든 종목을 반환하는 것이 정상 동작이며**, 특정 ETF 종목 처리 로직에 버그 존재
- API 서버 측 이슈로 확인되며, **테스트 후 즉시 배포 예정**

`include_trading_halted=true`의 동작이 제가 이해한 것(과거 거래 불가 종목 데이터 포함)과 다른 것 같아 문의드립니다.

- 497880은 거래 허용 종목이라 조회되지 않습니다.
- 내부적으로 파라미터 명이 혼동될 수 있어 개선 예정이며, 하위호환성 문제는 없음.
- 원하시면 `include_trading_halted=true` 시 거래 가능/불가 모두 포함하도록 변경 가능합니다.
