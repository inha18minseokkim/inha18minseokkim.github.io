```java
enum Test{
    A(0),B(1),C(2),D(3);
    private final int value;

    Test(int value) {
        this.value = value;
    }
}
```

정해져있는 상수 값을 정해줌(내부적으로는 int 같은걸로 구현)
enum 내부의 값들 중 하나만 가질 수 있음(다른 값 안됨)
장점 : 필드가 가질 수 있는 값을 특정 범위로 compile time 제한 가능(Type Safety)
Type Safety 보장이 되지 않는다면 문제 생길 수 있음
0 - A, 1 - B, 2 - C, 3 - D 이런식으로 해놓으면 if문으로 분기처리할듯. 상당히 더러워지고 예측할 수 없는 값이 들어가는 경우 Tracing, 방어 힘듬
각 요소의 인스턴스는 딱 하나만 만들어짐 - enum을 사용하여 싱글톤 패턴 안전하게 구현 가능

```java
public static void main(){
	Arrays.stream(OrderStatus.values()).forEach(System.out::println);
}
```

equals 안쓰고 == 써도 됨. 오히려 equals 쓰면 nullException 뜰 수 있음.

## EnumSet, EnumMap

Key값이 Enum으로 올 수 있는 경우 EnumSet, EnumMap이 해시보다 성능이 좋음
