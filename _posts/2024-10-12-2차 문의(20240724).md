AACT-WS

```json
[
    {
        "type": "string_pattern_mismatch",
        "loc": [
            "path",
            "symbol"
        ],
        "msg": "String should match pattern '^(?:[A-Z]+:)?[A-Z]{1,5}(?:-[A-Z])?(?:-[A-Z])?$'",
        "input": "AACT-WS",
        "ctx": {
            "pattern": "^(?:[A-Z]+:)?[A-Z]{1,5}(?:-[A-Z])?(?:-[A-Z])?$"
        },
        "url": "https://errors.pydantic.dev/2.7/v/string_pattern_mismatch"
    }
]
```

그리고 특정한 옵션이  있는 증권들은 - 로 구분하여 들어오는데 해당 티커의 형식이 구글이나 야후파이낸스에서 찾았을때와 좀 형식이 다른 경우도 있습니다. ex) ABR-P-E ⇒ ABR-PE
  - 케이뱅크 내부에서 티커 정보는 딥서치 티커를 따를 예정이라 티커를 어떻게 분류하고 관리하시는지 간단한 설명 부탁드립니다.
3.
현재 케이뱅크에서  일 단위로 기준정보 수정과 가격정보 수신을 수행 할 예정입니다.
symbol이 필수가 아닌 경우 (예컨데 국내주식 가격) 한 번 호출하겠지만 symbol이 필수인 경우 기준정보에 대한 호출이 N회 발생시킬 예정입니다.
  - 따라서 특정 시간대에  종목 리스트를 가져와서 각각 종목에 대한 가격 리스트를 호출하는 작업을 수행할 예정입니다 (해외주식 가격 약 7천회, 국내주식 재무비율 약 4천회 , 재무제표 4천회  등) 
  - 해당 작업의 리스크에 대해 검토 부탁드립니다.
