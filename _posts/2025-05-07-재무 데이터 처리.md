## 이슈 및 검토 사항

**1. 종목 목록 조회 API 대체 방안 미확보**

`GetStockSymbols(entities="")` API의 대체 수단이 현재 없는 상태로, 해결 방안 검토 필요.

**2. 재무제표 노출 기준 미정립**

어떤 재무제표 유형(연결/별도, 분기/연간, 누적/증분)을 기본으로 노출할지 기준이 확정되지 않은 상태.

**3. 간편 재무 요약 API 검토**

아래와 같은 형태의 간편 재무 요약 API를 활용하면 주요 지표를 한 번에 조회할 수 있다.

```json
{
  "symbol": "023760",
  "name_ko": "한국캐피탈",
  "reported_currency": "KRW",
  "fiscal_end_date": "2024-12-31",
  "revenue": "460175821000",
  "gross_profit": null,
  "operating_profit": "94600462000",
  "net_income": "81136484000",
  "asset": "4459495877000",
  "liability": "3785810414000",
  "equity": "673685464000"
}
```

다만 IFRS 별도 기준으로만 데이터를 제공하는 한계가 있어, 연결재무제표 기준이 필요한 경우에는 별도 처리가 필요하다.

**4. 업종 구분 코드(fs_type) 정의**

```
00=제조, AA=은행, BB=증권, CC=생보, DD=손보,
EE=신용금고, FF=종금, GG=투신, HH=리스,
II=카드, JJ=창투, KK=할부금융, ZZ=기타
```

**5. 신용카드업(II) enum 누락**

신용카드업(II) 코드가 애플리케이션 enum에 등록되어 있지 않은 상태. 개발 시 enum 추가 처리 필요.

**6. 매출액 데이터 부재 및 API 버그 확인**

일부 업종에서 매출액(영업수익) 데이터가 API 응답에 포함되지 않는 경우가 있었다. 데이터 공급처 웹사이트에서는 정상 표시되는 항목임을 확인하고, 아래 내용으로 문의를 진행했다.

> **문의 내용 요약**
> 삼성전자 24년 4분기 연결 재무제표를 `is_accumulated=false`와 `is_accumulated=true`로 각각 호출했을 때, 손익계산서 유량 항목의 값이 동일하게 반환됨. `is_accumulated=false`는 분기 변동분, `is_accumulated=true`는 회계연도 누적분이 각각 반환되어야 하는 것으로 이해했는데, 동작이 동일한 것이 올바른 스펙인지 확인 요청.

> **답변 요약**
> DB에 누적 데이터로 저장되어 있는데, `date_range`를 정확히 입력하면 해당 날짜의 누적 데이터만 조회되어 비누적 분기 데이터를 별도로 계산하지 못하는 버그가 확인됨. 수정 후 해당 주 테스트, 차주 초 릴리즈 예정이라는 답변을 받음.

**7. 연결 / 별도 재무제표 코드 체계 불일치**

연결재무제표와 별도재무제표 간에 업종 코드 기준이 일부 상이하므로, 구현 시 코드 분기 처리 필요.
