# 의문점, 해야할일

1. 현재 GetStockSymbols(entities="") 이 api 대체제가 없는상황임
2. 현재 재무제표 보여주는 기준 정립이 되어있지 않음
3. [https://api-kbank.deepsearch.com/v2/companies/KRX:023760/financials?date_from=2023-01-01&date_to=2025-03-09](https://api-kbank.deepsearch.com/v2/companies/KRX:023760/financials?date_from=2023-01-01&date_to=2025-03-09)
이 api를 사용하면 재무정보 잘 보여줌

4. 이거에 따라 재무제표 다름
fs_type
재무제표구분 (00=제조, AA=은행, BB=증권, CC=생보, DD=손보, EE=신용금고, FF=종금, GG=투신, HH=리스, II=카드, JJ=창투, KK=할부금융, ZZ=기타)
5. 내가 처음에 개발할 때 잘못한건지 딥서치가 잘못한건지는 모르겠는데 카드업(II)가 enum에 등록이 안되어있음. 개발할 때 전환 필요
6. 그렇다고 재무제표 풀데이터에 매출액,영업이익,당기순이익이 다 있는 구조는 아닌 것 같음.
  - 매출액(영업수익)의 경우 일부 데이터가 보이지 않음. 딥서치에서 취급을 안하는거라기에는 웹사이트상에는 잘 나와있음 ㅇㅇ
