신규할때 다시 논의

서비스코드(apigateway only)
  - MCI → ingress(SvcCd) → apigateway(SvcCd to URI) → controller(URI)
  - 현재 BXM DB에 매핑정보가 저장되어서 MCI에서 코드를 받으면 해당 클래스를 콜하는 구조임
  - 1안) apigateway에 enum 정의
  - 2안) db에 저장 → 관리자 화면 정의해야함
⇒ api-gateway에 enum 넣기로 함. 당장 db에 api gateway 디펜던시가 생기는것이 좋지않다고 판단함(사실 로깅하기 위해서 이미 의존이 있긴 한데 나중에 떼야함 이건)

URI 국제표준으로 사용
  - /도메인서비스/버전/서브도메인/콜렉션/리소스
현재 MCI에서 post body에 데이터를 던져주는방식(안좋은방식)이라 PathVariable은 사용 불가.
MCI 에서 body를 map으로 받으면 k:v를 풀어서 queryParam형식으로 줌
