### 2차

**안녕하세요 데이터서비스팀 임태훈입니다.**
**추가로 api 2건 데이터 적재가 가능할지 메일드립니다..**
**해당 두건은 주기적(일별)으로 업데이트되는 데이터테이블이 아닌 메타테이블입니다..**
**따라서 Api 파라미터에 날짜 값이 따로 들어가지 않습니다.**
**Tag 메타데이터 테이블의 row수가 현재기준 234개 컬럼이 5개 정도이며,**
**Tag Find테이블의 경우 각 Tag메타데이터에 1:many 의 관계로 몇 천건씩 존재하고 있습니다.**
- **Tag 메타데이터**
**n   가이드 : https://help.deepsearch.com/dp/api/func/industry/subject/getconditionaltagmetadata**
**n   URL : https://api.deepsearch.com/v1/compute**
**n   Params : input = GetConditionalTagMetadata( )**
- **Tag find 데이터**
**n   가이드 : https://help.deepsearch.com/dp/api/func/industry/subject/findentitybytag**
**n   URL : https://api.deepsearch.com/v1/compute**
**n   Params : input = FindEntityByTag(tag_uids = ‘ tag 메타데이터에서 받아온 tag =uids 값 ’)**
**(기존에 전달드렸던 기업배당내역/ 기업제무재표 / 거개량, 시가총액, 종가 등 / 기업배수 관련데이터 / 뉴스 / 증권사리포트**
**위 api 들의 경우에는 매일 일별로 전일자(d-1) 데이터를 DB에 쌓아두길 희망합니다.)**
**노고에 감사드리며, 필요하신 정보나 추가로 확인이 필요한 내용은 말씀부탁드립니다.**
**임태훈 드림.**
