1. 원하는 포맷은 JSON 형식이며, 데이터 예시는 다음과 같다고 예상하고 있습니다

```json
{
"list": [{
		"기준일시" : yyyymmddHHmmss,
		"종목코드번호" : 279570,
		"주식명" : "케이뱅크",
		"장외정보" : "일반",
		"조회건수" : 333333,
		"추정가치" : 333333333333333,
		"언론가치" : 222222222222,
		"전일기준가격" : 20000,
		"전일최대가격" : 30000,
		"52주(연간)최대가격" : 30000,
		"전일최소가격" : 10000,
		"52주(연간)최소가격" : 10000,
		"전일자대비변동가격" : -500,
		"금일변동비율" : -0.233,
		"원링크URL" : "www.google.com",
		"종목상태값" : "#"
	}]
}
```

  - 매일 위와 같은 포맷의 데이터를 받는것을 기대하고 있습니다
  - 두나무 기준 영어 필드 이름을 정확하게 알지 못하여 자체적으로 한글 용어를 사용한 점 양해 부탁드립니다.

2. 첫 데이터 수신 api 개발 완료 후 대략 90일 정도의 데이터는 파일로 제공해 주실 수 있으실까요? 초기 적재 수행을 위해 필요할 것 같습니다.
저희는 증권식별번호에서 채번한 종목코드번호로 특정 종목을 식별할 예정이므로, 다음과 같은 이슈를 예상하고 있습니다
1. 매일 저희가 데이터를 수신할 예정인데, 특정 주식이 상장하거나 증권이 소각되는 경우, 데이터에서 더이상 나타나지 않는지 문의드립니다(추가된 종목이 없다는 가정하에, 케이뱅크가 상장하는 경우 제공되던 데이터가 2200개 → 2199개)
2. 매일 정보가 업데이트 되지 않는 종목의 경우, 특정 필드의 값이 어떻게 들어오는지 문의드립니다
  - (가령, 2월 29일 케이뱅크 주식이 거래되지 않았다면 
    - a. 데이터가 들어오지 않는지 
    - b. 2월 29일 0원으로 들어오는지 
    - c. 2월 29일에 2월 28일 이력인 20000원(?) 으로 들어오는지
    - d. 기준일자가 2월 28일인 데이터가 들어오는지
요건이 제한되어 있지 않고 열려있다면, c 형태를 선호합니다. 또는 종목상태값에서 Update, Delete, Insert를 제공해주셔도 무방합니다.
