상장 전 투자서비스 제휴 목적
상장,금투 등등


- 종목 차트 탭(api를 세개 나눌지 한번에 줄지 물어봐야함)
  - 기준가가 높은
  - 조회수가 높은
  - 예상 시총이 높은
- INPUT
  - 기준일자
  - 조회조건(?)
- OUTPUT
  - 기준일자
  - 리스트카운트
  - 리스트
    - 증권식별번호(아마종목코드)
    - 종목명
    - 기준가
    - 전일대비 등락률(부호포함)
    - 조회수
    - 관심여부
    - (?) 누적조회수
    - (?) 예상시총
페이징처리 필요여부(총 몇건 보여줄 것인지)
  - 한 10~20개정도 생각중
API를 세개로 할지 하나로 할지 
전일대비가격 없으면 어떻게 할지


- 관심 종목 탭
  - INPUT
    1. 고객식별자
    2. 기준일자
    3. 어떤순서로할지코드
  - OUTPUT
    1. 리스트카운트
    2. 종목리스트
      1. 종목명
      2. 일반/전문종목
      3. 종목코드(6자리)
      4. 현재가격
      5. 전일대비등락가격(부호포함)
      6. 전일대비등락비율(부호포함)
      7. 전일최고가
      8. 52주최고가
      9. 전일최저가
      10. 52주최저가
      11. 관심종목여부
  - 저 이미지의 정체는 무엇인지(7’)
    - 90일자 기준


- 관심종목등록
  - INPUT
    1. 고객식별자
    2. 증권식별번호(종목코드로 대체가능?)
  - OUTPUT
    1. 처리개수

- 종목 세부
  - INPUT
    1. 증권식별번호(종목코드)
    2. 기준일자
    3. 고객식별자
    4. 조회조건(1개월/3개월, 한 번에 줄지 여러번에 줄지에 따라 다름)
  - OUTPUT
    - 일반전문종목여부
    - 관심종목여부
    - 기준일자
    - 종목코드
    - 종목명
    - 현재가격(맞음?)
    - 전일종가대비등락률
    - 전일최고가
    - 전일최저가
    - 52주최고가
    - 52주최저가
    - 추정가치
    - 언론에서본기업가치
    - n개월평균가격(1/3개월 기준)
    - n개월최고가격
    - n개월최저가격
    - n개월 리스트카운트
    - 증권플러스URL
    - 가격리스트
      - 기준일자
      - 기준가격
  - 위의 7’ 14200과 밑의 기준가 14700의 차이는 무엇인가
  - 1개월 3개월api를 쪼개줄지 한번에 리스트로 넘길지


- 종목 검색(2200개 정도 되는 리스트 채널에서 필터링할건지 서버에서 검색돌릴지에 따라 달라짐)
  - INPUT
    - 기준일자
    - 검색텍스트
  - OUTPUT
    - 뭘까(화면미정의)
밑에 있는 전체 인기종목은 오늘 조회기준을 말하는것?
최근 7일 이내 조회수 누적을 데이터로 받는데 이건 행내 클릭수를 보는게 아니라 바깥의 클릭수를 가져오는것? 아니면 우리가 조회수를 따로 수집할 것인지?
⇒ 우리 조회수는 집계하지 않고 증권플러스 기준의 조회수를 사용하는것으로 결정.

    3. 종목리스트
      1. 증권식별번호
      2. 정렬순서
  - OUTPUT
